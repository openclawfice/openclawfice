# GitHub Repo Setup (30 Seconds)

**Why:** Repo has NO description and NO topics. This kills discoverability on GitHub search.

**Impact:** HIGH - GitHub is the #1 discovery channel for developer tools

---

## Quick Setup (30 seconds)

### Step 1: Go to Settings
https://github.com/openclawfice/openclawfice/settings

### Step 2: Add Description
**Description:**
```
Your AI agents, but they're Sims 🏢 Pixel art virtual office for OpenClaw agents
```

**Website:**
```
https://openclawfice.com
```

**Check these boxes:**
- ✅ Releases
- ✅ Packages
- ✅ Deployments

### Step 3: Add Topics (scroll down to "Topics")
Click "Add topics" and add these (one at a time):

```
openclaw
ai-agents
pixel-art
dashboard
office
retro
sims
agent-management
automation
developer-tools
nextjs
typescript
open-source
```

**Click "Done"** when finished.

### Step 4: Verify
Refresh the repo homepage - you should see:
- Description at the top
- Topics as clickable pills
- Website link next to About section

---

## Why This Matters

**Before (bad):**
- Repo shows up as blank/unnamed in search
- No topics = no discovery
- Looks abandoned/unprofessional

**After (good):**
- Clear value prop visible immediately
- Shows up in GitHub topic searches
- Looks polished and trustworthy
- People click through to website

---

## Bonus: Approve Vercel GitHub App

While you're in settings:
1. Go to https://github.com/orgs/openclawfice/settings/installations
2. Find "Vercel" in pending installs
3. Click "Configure" → "Approve"

This lets Vercel auto-deploy from GitHub (currently pending).
