## Three Critical Fixes

### 1. Early Node.js Version Check ✅

**File:** `bin/openclawfice.js` (lines 10-21)

**Problem:**
Users run `npm install`, wait 30-60 seconds downloading packages, then hit cryptic error about Node compatibility. Wasted 2-3 minutes.

**Solution:**
Check Node version BEFORE anything else runs:

```javascript
const nodeVersion = process.versions.node;
const nodeMajor = parseInt(nodeVersion.split('.')[0], 10);
if (nodeMajor < 18) {
  console.error('\n❌ Node.js version too old!\n');
  console.error(`You have: v${nodeVersion}`);
  console.error('Required: v18.0.0 or newer\n');
  console.error('Update Node.js:');
  console.error('  macOS/Linux: https://nodejs.org/en/download/');
  console.error('  Or use nvm: nvm install 22 && nvm use 22\n');
  process.exit(1);
}
```

**Impact:**
- Fails in <1 second (not 3 minutes)
- Shows current vs required version
- Gives 2 upgrade paths (official download + nvm command)
- User knows exactly what to do

### 2. Actionable OpenClaw Error ✅

**File:** `bin/openclawfice.js` (lines 25-38)

**Before (Generic):**
```
❌ OpenClaw not found!

OpenClawfice requires OpenClaw to be installed and configured.
Expected config at: ~/.openclaw/openclaw.json

Visit https://openclaw.ai to get started.
```

**Problems:**
- "Visit site" is vague - what do I do there?
- `~` confuses Windows users and non-technical users
- No copy-paste command
- Doesn't explain "new terminal" requirement

**After (Actionable):**
```
❌ OpenClaw not installed!

OpenClawfice needs OpenClaw running first.

📍 Where I looked:
   /Users/tyler/.openclaw/openclaw.json
   (~ means "/Users/tyler")

🔧 How to fix:
   1. Install OpenClaw:
      curl -fsSL https://openclaw.ai/install.sh | bash
   2. Wait for it to finish
   3. Open a NEW terminal window
   4. Run this command again

💡 Need help? https://docs.openclaw.ai/installation
```

**Impact:**
- Shows actual expanded path (no ~ confusion)
- Copy-paste install command (one-liner)
- Numbered steps - clear sequence
- Explains WHY new terminal (PATH reload)
- Links to docs for deeper help

### 3. Discovery Animation Bug Fix ✅

**File:** `app/page.tsx` (line 315 removed)

**Problem:**
Two `setShowDiscovery(true)` calls:
1. Line 293: Conditional (checks localStorage for first-run flag)
2. Line 315: UNCONDITIONAL (always fires)

The second call overrode the first, so animation triggered EVERY page load instead of just once.

**Solution:**
Removed the duplicate unconditional call. Now animation:
- ✅ Fires once per browser on first agent load
- ✅ Respects localStorage flag (`openclawfice_discovery_seen`)
- ✅ Never repeats after user has seen it

**User Experience:**

   - Terminal green background (#00ff41)

   - Audio plays (`/sounds/discovery.mp3` at 30% volume)
4. Animation completes after 1.5 seconds
5. Office fades in
6. **Screenshot moment** - worth sharing

**Why It Matters:**
